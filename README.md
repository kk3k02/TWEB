## üéØ Mandatory Sections Implementation

[cite_start]The project includes all five mandatory sections with the required content and functionality[cite: 14]:

### 1. Conference (Home Page)

* [cite_start]**General Info:** Location, dates, main theme, and past editions are presented[cite: 15].
* [cite_start]**Committees:** Organizing Committee and Technical Committee (simulated data) are listed[cite: 15].
* **Sponsors:** Major sponsors and partners are showcased.
* [cite_start]**Image Gallery:** A gallery section features simulated archival images from past events/presentations[cite: 16].

### 2. Program

* [cite_start]**Schedule:** A detailed daily program is provided, including regular sessions, special sessions, and social events[cite: 17].
* [cite_start]**Themes:** All main conference themes are presented in a dedicated section[cite: 17].
* [cite_start]**Keynote Speakers:** Profiles of the keynote speakers are featured in a grid layout[cite: 18].

### 3. Paper Submission

* [cite_start]**Information:** Details on important dates, submission procedures, and paper templates are available[cite: 19].
* [cite_start]**Contact Form:** Includes a dedicated form to contact the organization[cite: 20].
    * [cite_start]**Fields**: Collects name, gender, ID card number, address, email, and phone number[cite: 21].
    * [cite_start]**Reason Select**: Allows the user to select the reason for contact (Paper Submission, Registration, Other Questions)[cite: 22].
    * [cite_start]**Conditional Input**: If "Other Questions" is selected, an additional text input appears to specify the topic[cite: 23].

### 4. Registration

* [cite_start]**Pricing:** Details on event prices and registration procedures are provided[cite: 24].
* [cite_start]**Logistics:** Information regarding location/facilities, transportation, and contacts is included[cite: 25].
* [cite_start]**Touristic Material:** Features a slideshow element showcasing local touristic material related to the location[cite: 26].

### 5. Membership

* [cite_start]**Login/Registration:** This section allows existing users to log in and new users to register[cite: 27].
* [cite_start]**Simulated Authentication:** The entire authentication/registration process is simulated using Vanilla JavaScript (client-side)[cite: 28].
